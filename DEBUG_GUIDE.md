# 前后端交互问题排查指南

## 问题排查步骤

### 第一步：检查后端服务是否运行

1. **验证后端服务端口**
   ```bash
   # Windows PowerShell
   netstat -ano | findstr :8081
   
   # 或直接在浏览器访问
   http://localhost:8081/poi-type/list
   ```

2. **检查后端控制台**
   - 查看是否有启动日志
   - 查看是否有错误信息（红色）
   - 查看是否有 SQL 查询日志

3. **常见问题**
   - 端口被占用
   - 数据库连接失败
   - Java 进程未启动

---

### 第二步：检查前端请求是否发出

1. **打开浏览器开发者工具**
   - 按 F12 或右键 → 检查
   - 切换到 Console（控制台）标签
   - 切换到 Network（网络）标签

2. **在控制台查看日志**
   - 应该看到 `🚀 发送请求:` 开头的日志
   - 应该看到 `📡 开始请求 POI 列表` 日志

3. **在 Network 标签查看请求**
   - 点击 Filter，选择 XHR/Fetch
   - 查看是否有 `/api/...` 的请求
   - 查看请求的状态码（Status）

---

### 第三步：验证代理配置

Vite 代理配置在 `vite.config.js`：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8081',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

**实际请求流程：**
1. 前端发起: `GET /api/poi/of/type?typeId=1`
2. Vite 代理转发: `GET http://localhost:8081/poi/of/type?typeId=1`

---

### 第四步：数据库连接检查

**后端可能的问题：**
1. **数据库未启动**
   - MySQL 服务是否运行
   - PostgreSQL 服务是否运行

2. **连接配置错误**
   - 检查 `application.yml` 或 `application.properties`
   - 验证数据库地址、端口、用户名、密码

3. **连接池配置**
   - 检查连接池是否耗尽
   - 查看超时设置

---

## 常见错误及解决方案

### 错误 1：无法连接到服务器 / ECONNREFUSED

**现象：**
- 控制台显示 `❌ 无法连接到服务器`
- Network 中请求显示红色 (failed)

**排查：**
1. 后端服务是否启动
2. 端口是否正确（8081）
3. 防火墙是否阻止

**解决：**
```bash
# 检查端口占用
netstat -ano | findstr :8081

# 如果被占用，杀掉进程
taskkill /PID <进程ID> /F
```

### 错误 2：请求超时 / ECONNABORTED

**现象：**
- 控制台显示 `❌ 请求超时，请检查网络`
- 页面持续加载

**排查：**
1. 数据库查询是否过慢
2. 后端接口是否有死锁
3. 网络是否正常

**解决：**
- 检查后端 SQL 查询性能
- 增加索引优化查询
- 检查是否有长时间运行的事务

### 错误 3：404 接口不存在

**现象：**
- 控制台显示 `❌ 请求的资源不存在`
- Network 显示 404 状态码

**排查：**
1. 接口路径是否正确
2. Controller 注解是否正确
3. 包扫描是否包含该 Controller

**解决：**
- 检查前端请求路径和后端 `@RequestMapping` 是否匹配

### 错误 4：401 未授权

**现象：**
- 自动跳转到登录页面
- 控制台显示 `❌ 登录已过期`

**排查：**
1. localStorage 中是否有 token
2. token 是否过期
3. 后端是否正确验证 token

**解决：**
- 重新登录获取新 token

## 快速诊断清单

使用这个清单快速定位问题：

- [ ] 后端服务正在运行（端口 8081）
- [ ] 后端控制台没有报错
- [ ] 数据库连接正常
- [ ] 浏览器控制台有 `🚀 发送请求:` 日志
- [ ] Network 面板中有请求发出
- [ ] 请求的 URL 正确
- [ ] 请求参数完整
- [ ] 后端收到请求（有日志）
- [ ] 后端返回了响应（有日志）
- [ ] 前端收到了响应（有 `✅ 收到响应` 日志）

---

## 临时测试方案

如果需要快速测试后端是否正常，可以：

1. **直接在浏览器访问**
   ```
   http://localhost:8081/poi-type/list
   ```

2. **使用 curl 测试**
   ```bash
   curl http://localhost:8081/poi-type/list
   ```

3. **使用 Postman 或 API 调试工具**
